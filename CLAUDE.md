## 4. Операции

### OP-1: Daily Review
Триггер: "ревью", "вечернее ревью", "что я сделал".

**Дата ревью (D):** по умолчанию = сегодня. Если запрос утром о вчера — D = вчера. Если явно указана дата — берёшь её.

**Три независимых источника сигнала:**
```
1. git log --since="D 00:00" --until="D 23:59" --name-only --diff-filter=AMR
   → фильтруй пути: resource/, areas/, projects/
   → это "что реально изучал/писал" (даже без diary)
2. temporary/daily/D.md — читаешь если существует
3. Calendar: list_events(D) — только учебные события (исключи работу, дорогу, gym)
```

**Решение о создании ревью:**
- SKIP только если все три источника пустые → одна строка в `_claude/log.md`
- Если есть git-активность ИЛИ diary ИЛИ учебные события → создаёшь `_claude/review/D.md`

**Два формата ревью:**

| Условие | type | Содержание раздела "Факты" |
|---------|------|----------------------------|
| Есть diary | `daily-review` | План (calendar) vs факт (diary + git) |
| Нет diary, есть git/calendar | `sparse-review` | Активность по git + флаг "diary missed" |

**Шаги:**
```
4. Сравни план (calendar) vs факт (git + diary) → таблица в ревью
5. Если sparse-review — реконструируй темы по diff изменённых файлов
6. Notes Audit: для КАЖДОГО изменённого .md в resource/ /areas/ /projects/ —
   читай diff (или весь файл если новый) и проверяй:
   - ❌ фактические ошибки (формулы, определения, неверные утверждения)
   - 🔀 противоречия с другими конспектами в vault (ищи через grep по ключевым терминам)
   - 📐 структура (соответствует ли шаблону §5: frontmatter, "Выжимка", "Формулы", "Примеры", "Связанные")
   - 📝 полнота (пустые секции, оборванные предложения, "TODO", "...")
   - 🏷️ терминология (путаница в обозначениях, несогласованные имена переменных)
   Замечания добавь в раздел "Notes Audit" ревью (по файлу — пункт со списком проблем).
   Если файлов >5 — приоритезируй по новизне (сначала созданные, потом изменённые).
7. Запиши в _claude/review/D.md
8. Обнови _claude/log.md (одна строка)
9. Если есть систематические срывы или критичные ошибки в конспектах → скорректируй Calendar на завтра / добавь TODO в _claude/notes.md
```

**Notes Audit — формат раздела в ревью:**
```markdown
## Notes Audit

### resource/CV/image-interpolation.md (новый, 120 строк)
- ❌ Формула билинейной интерполяции: индексы перепутаны (см. строка 47)
- 📐 Нет секции "Связанные концепты" — добавить ссылки на [[image_convolution]], [[image-sampling]]
- 📝 Раздел "Примеры" пуст

### resource/machine_learning/vision_transformer.md (обновлён, +35 строк)
- 🔀 Противоречит [[language_transformer]]: разные обозначения для positional encoding
- 🏷️ В одном абзаце "patch size" обозначен P, в другом — p. Унифицировать.
```

**Формат лог-записи:**
- `## [D] review | <тип> | <Nч учёбы> | <темы через запятую>`
- `## [D] skip | пусто (git/diary/calendar)` — только если все три пустые

**Sparse-review шаблон (раздел "Факты дня"):**
```markdown
## Активность (реконструкция по git)
**Изменённые файлы:**
- resource/CV/image-interpolation.md — создан (~120 строк)
- resource/machine_learning/vision_transformer.md — обновлён

**Реконструированные темы:** [читаешь diff, извлекаешь концепты]

**Флаг:** 📌 Diary не заполнен — рекомендую 3-строчную запись.
```

### OP-5: Lint
Триггер: "lint", "проверь вики".
