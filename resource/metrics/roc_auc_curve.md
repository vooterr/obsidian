## Вычисление
Вычисляется путем создания кривой, где 
	- По оси OY - **TPR**(TP) - доля верно классифицированный объектов
	- По оси OX - **FPR**(FP) - доля неправильно классифицированных объектов
- **Идеальной кривая** - будет в случае прямой линии, проходящей по прямой y=1
- **Худшая кривая** - будет в случае линии проходящей по прямой x=1
- ![[roc_auc_curve.png]]
- **AUC** - это площадь под кривой ROC, которая служить количественной оценкой, классификатора(чем выше тем лучше)
	- AUC с учетом ограничения графика принимает значения $0 \leq AUC \leq 1$
### В каких целях используется
- **Оценка модели в целом**. Величина AUC позволяет сравнить качество разных алгоритмов независимо от того, какой конкретный порог классификации вы выберете
- **Поиск баланса качества**. Двигаясь по кривой, можно задать идеальный порог срабатывания алгоритма, исходя  из *бизнес задачи* или *цены ошибки*, например *лучше не выписывать здоровому человеку лишний раз антибиотики, чем не выписать лишний раз туберкулезнику*
### Что делать при неравномерном распределении классов
- *При сильном дисбалансе классов* (например, когда искомое событие встречается у 1 объекта из миллиона) метрика ROC-AUC может *давать излишне оптимистичную оценку*
- Есть несколько решений, при неравномерном распределении классов
	1. **Переход к PR-кривой (ТП-кривой) и AUC-PR.** Вместо ROC-кривой лучше строить кривую соотношения точности (precision) и полноты (recall). Эти метрики фокусируются именно на качестве нахождения объектов редкого целевого класса и не искажаются из-за большого количества истинно отрицательных ответов. Для сведения их к одному числу также используют F-меру
	2. **Искусственная балансировка выборки.** Обучающую выборку формируют неслучайным образом, чтобы искусственно уравнять количество объектов всех классов (например, удаляя часть объектов преобладающего класса)
	3. **Введение весов классов (cost-sensitive learning).** Модели задают разные величины потерь (штрафы λ) за ошибки первого и второго рода. Пропуск редкого, но важного события наказывается гораздо строже, чем ложное срабатывание на объекте частого класса
