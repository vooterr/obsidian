## Что за метрика

- Метрика, которая совмещает в себе несколько других метрик, таких как *precision(точность)*, *recall(полнота)*
- В отличии от ROC AUC **может корректно выдавать результат в несбалансированных классах**
### Precision (Точность)

- **Из всех, кого модель назвала положительными — сколько действительно положительные?**

- $$Precision = \frac{TP}{TP + FP}$$
- *Пример:* спам-фильтр пометил 100 писем как спам. 90 из них действительно спам → Precision = 0.90     
### Recall (Полнота)

- **Из всех реально положительных — сколько модель нашла?**
- $$Recall = \frac{TP}{FP + FN}$$
## Компромисс (Trade-off)

- Precision и Recall находятся в **противоречии** друг с другом. Он контролируется **порогом классификации** (threshold):
	- **Повышаем порог** → модель осторожнее → больше Precision, меньше Recall
	- **Понижаем порог** → модель агрессивнее → больше Recall, меньше Precision
## PR-кривая и PR AUC

- **PR-кривая** строится путём изменения порога классификации от 0 до 1 и записи значений (Recall, Precision) в каждой точке. Каждая точка на кривой — это один порог.
- **PR AUC** (Area Under PR Curve) — площадь под этой кривой. Чем больше — тем лучше:
	- **PR AUC = 1.0** — идеальная модель
	- **PR AUC = доля позитивных классов** — случайная модель (random baseline на графике)
