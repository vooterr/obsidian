## Описание
- В отличии от градиентного спуска, где градиент вычисляется по всей выборке, *стохастический градиент* - для минимизации *эмпирического риска*, берется случайно часть из выборки (один или пачка объектов), и на нем вычисляется градиент
	- По закону больших чисел, через большое количество итераций, результат достигнет какого то довольно хорошего значения
- Если смотреть на движение весов в n-мерном пространстве, то движения градиентов будет похоже на броуновское движение
	- Траектория оптимизации будет немного шататься 
	- Но в среднем двигаться в сторону убывания ошибки
### Алгоритм
1. Инициализация весов (можно применить некоторые эвристики)
2. Цикл (пока не сойдется или через k итераций)
	1. Выбирается случайный объект $x_{i}$ из выборки
	2. Вычисляется ошибка модели на этом объекте (*значение функции потерь и ее градиент*)
	3. Вектор обновляет веса по формуле
		$$w := w - \eta \nabla L_{i}(w), \text{где } \eta \text{ - темп обучения, а } \nabla L_{i}(w) \text{ - градиент функции потерь на объекте } x_{i}$$
- Оценка функционала происходит с помощью формулы экспоненциального скользящего среднего 
### Плюсы и минусы
- Плюсы
	- На большое выборке данных, можно найти хорошее решение, не просматривая всю выборку 
	- Довольно легко программируется 
	- Динамическое обучение (*подходит когда модель необходимо обновлять каждый раз, при поступлении данных*) 
- Минусы
	- Возможна сходимость к локальному минимуму или медленная сходимость из за нечеткого следования градиенту 
	- Необходимо тщательно подбирать темп обучения 
	- Возможна неустойчивость классификации и резкий рост нормы вектора весов (особенно если признаков много, а объектов мало, или есть мультиколлинеарность).
## Эвристик и для улучшения метода
### Инициализация весов
- Рекомендуется инициализировать веса небольшими случайными значениями (например, из равномерного распределения в интервале $[- \frac{1}{2n}, \frac{1}{2n}]$, где n — размерность пространства признаков)
- Либо инициализация *нулем* **!ТОЛЬКО ДЛЯ ПРОСТЫХ ЛИНЕЙНЫХ МОДЕЛЕЙ!**
- Для линейных моделей также можно использовать начальное приближение основываясь на корреляции признака с целевым вектором
	- $w_{j} = \frac{\langle y_{i}, f_{i} \rangle}{\langle f_{i}, f_{i} \rangle}$   
- Для многослойных сетей лучше использовать постепенное обучение каждого слоя по отдельности 
### Порядок объектов
- **Перетасовка (Shuffling):** Объекты следует выбирать случайно или периодически перемешивать выборку. Рекомендуется попеременно предъявлять объекты разных классов, чтобы направление градиента не было смещено в сторону одного класса слишком долго
- **Частое предъявление трудных объектов:** Можно чаще выбирать объекты, на которых текущий алгоритм совершает большую ошибку. Однако эту эвристику нужно применять осторожно: если в данных есть «шумовые выбросы», алгоритм может сосредоточиться на их бесполезном выучивании
- **Пропуск простых объектов:** Если ошибка на объекте меньше заданного порога, веса не обновляются. Это ускоряет эпоху обучения, фокусируя ресурсы на пограничных объектах (аналог правила Розенблатта)
### Выбор и адаптация шага обучения($\eta$)
- **Убывание со временем**: Шаг $\eta$ должен убывать с каждой следующей итерацией, например $\eta_{t}  = 1/t$, чтобы процесс затухал при приближении к минимуму, а не катался вокруг него
- **Скорейший градиентный спуск**: на каждом шаге можно аналитически искать решение для шага $\eta$, решая задачу минимизации $Q(w - \eta \nabla Q) \rightarrow min$. Для кв функций потерь этом можно сделать аналитически
  - **Диагональный метод Левенберга-Марквардта**: это метод второго порядка, где шаг выбирается индивидуально для каждого веса $w_{jh}$ на основе кривизны функции потерь( второй производной ).
### Регуляризация
- Для борьбы с переобучением и мультиколлениарностью, вводятся регуляризаторы, которые штрафуют за большие веса модели
- **Сокращение весов**: Добавление $L_{2}$ регуляризатора $\frac{\tau}{2}||w||^{2}$ приводит к тому, что на каждом шаге веса уменьшаются на коэффициент ($1 - \eta \tau$) < 1. Веса постоянно усыхают, стремясь к нулю, если их не поддерживают градиенты от данных. Это предотвращает неограниченный рост и паралич сети 
- **Отбор признаков(Lasso)**: использование $L_{1}$ регуляризатора приводит к занулению весов у неинформативных объектов 
### Метод инерции
- Чтобы уменьшить "шараханье" из стороны в сторону, усредняют градиенты по последним итерациям. Это позволяет двигать более целенаправленно
