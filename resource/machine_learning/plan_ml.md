## 🛠 Appendix: Таксономия работы с данными (Data Operations)

Методы обработки данных для `sklearn` пайплайнов.

### 1. Очистка (Cleaning)
* **Imputation (Заполнение пропусков):**
    * `SimpleImputer`: Среднее/Медиана (для нормальных/скошенных данных).
    * `KNNImputer`: Заполнение на основе похожих объектов.
* **Outliers (Выбросы):**
    * *Z-score:* Удаление, если $|x - \mu| > 3\sigma$.
    * *IQR:* Удаление за пределами $[Q1 - 1.5IQR, Q3 + 1.5IQR]$.

### 2. Масштабирование (Scaling)
*Обязательно для: Линейных моделей, KNN, Нейросетей. Не нужно для: Деревьев.*
* `StandardScaler`: $z = \frac{x - \mu}{\sigma}$. Сохраняет выбросы, приводит к $N(0,1)$.
* `MinMaxScaler`: $x_{new} = \frac{x - min}{max - min}$. Сжимает в $[0, 1]$. Чувствителен к выбросам.
* `RobustScaler`: Использует медиану и квартили. Устойчив к выбросам.

### 3. Преобразование (Transformation)
*Цель: Приблизить распределение к нормальному (Гауссову).*
* `Log Transform` ($\log(1+x)$): Для данных с длинным хвостом (цены, зарплаты).
* `Box-Cox / Yeo-Johnson`: Автоматическая параметрическая нормализация.
* `Discretization` (Bining): Превращение непрерывной величины в категории (бины).

### 4. Кодирование (Encoding)
*Цель: Перевод категорий (строк) в числа.*
* `One-Hot Encoding`: Создает бинарные столбцы. (Хорошо для: Линейных моделей. Плохо: Если категорий > 100).
* `Label Encoding`: Замена на числа 1, 2, 3... (Только для порядковых данных: Low, Medium, High).
* `Target Encoding`: Замена категории на среднее значение таргета. (Риск переобучения! Требует регуляризации).

### 5. Отбор признаков (Feature Selection)
* **Filter methods:** Корреляция (Pearson), VIF, $F$-test.
* **Wrapper methods:** RFE (Recursive Feature Elimination).
* **Embedded methods:** L1-регуляризация (Lasso), Feature Importance из деревьев.
