# SGD с моментом

## 📝 Основное содержание
- Обычный SGD имеет несколько недостатков
	- Его движение имеет хаотичный характер, двигает он по броуновскому движению, из стороны в сторону
	- *Он почти не двигает* в местах где *плоскость* функции потерь *практически плоская*, поэтому в этих местах может замедляться
- Но есть *SGD с моментом движения*, который гасит эти недостатки, позволяя "шарику набирать инерцию"

## 🔗 Связи

## Движение SGD с моментом
- Движение SGD с моментом имеет более плавный характер, потому что он запоминает движения, в которых точка двигалась до этого
- Применяя эти знания, каждое движение алгоритма имеет последствие в будущем, поэтому каждый толчок в сторону от движения будет чувствоваться меньше
	- Поэтому хождения влево и вправо будут глушить друг друга, а движения "шарика" будет инертным
## Алгоритм в действии
- Для понимания алгоритма, необходимо ввести новую переменную: $v_{i}$ - вектор скорости(velocity), при помощи которого обновление будет теперь двухшаговым:
	- $$\begin{array}{l} v_{t}=\beta * v_{t - 1} + (1 - \beta) * g_{t} \\ \theta_{t}= \theta_{t-1} - \eta * v_{t} \end{array}$$
- Обычно **параметр** $\beta$ - это **коэффициент трения**, которые указывает, *сколько процентов накопленной скорости ты сохраняешь для нового градиент*
	- Для значения $\beta = 0.9$ *мы сохраняем 90% прошлой скорости и лишь 10% будут составлять от нового градиента*
- Рассмотрим равенство для скорости при разложении рекуррентного соотношения: 
	- $v_{i}= (1 - \beta) * g_{t}+ \beta * (1 - \beta) * g_{t-1} + \beta^{2} * (1- \beta) * g_{t-2}\dots$ 
- Это будет *экспоненциально взвешенное скользящее среднее***EMA** всех прошлых градиентов, где самый свежий градиент имеет вес $(1 - \beta)$, второй - $\beta * \beta(1 - \beta)$ и так далее.
	- **Старые градиенты экспоненциально забываются** 
	- Тогда,* эффективное окно памяти рвано примерно* $\frac{1}{1 - \beta}$ шагов, или примерно 10 шагов

### ## Зачем именно экспоненциальное взвешивание, а не простое среднее?

- Хороший вопрос, который стоит задать явно. Можно было бы взять простое среднее последних `k` градиентов — это тоже сгладило бы шум. Но у EMA два практических преимущества.
	1. Во первых, это преимущество по памяти: для простого среднего, необходимо хранить `k` средних значений, когда для **EMA** достаточно хранить только вектор скорости
	2. Во вторых, забывание в *простом среднем, градиент, которые были до, ценятся также как и нынешний градиент *
		- То есть нету никакого забывания, есть линия которая со временем немного меняется
### ## Феномен перелёта: почему большой β опасен

Когда ты поставишь `β=0.99` в демо, ты заметишь: в начале loss падает быстро, но потом начинает... колебаться или даже расти. Это **overshooting** — «перелёт».
