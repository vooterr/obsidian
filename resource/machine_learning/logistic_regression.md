## Для чего
- Логистическая регрессия - это *один из самых популярных алгоритмов классификации* в машинном обучении
- Логистическая регрессия - это мост между *теорией восстановления регрессии* и *задачами дискретной классификации*.
- Логистическая регрессия - метод построения классификатора, позволяющий *оценивать апостериорные вероятности* принадлежности к классу
	- Проще говоря, *выдает ответ и вероятность данного ответа*
### Постановка задачи
- Пусть дан какой то тренировочный датасет: ${(x_{i}, y_{i})}^{n}_{i=1}, x_{i} \in \mathbb{R}^{k}, y_{i} \in {C_{1}, \dots C_{K}}$ 
- Модель $c(x)$ предсказывает класс или вероятность принадлежности какому либо классу
- Функция потерь $L(X, y, f)$ должна быть минимизирована 
## Понятие отступа(margin)
- Определение: $M_{i} = y_{i} \langle w, x_{i}\rangle$ (произведение правильного ответа на предсказание модели до сигмоиды)
- Смысл: чем больше $M_{i}$ те больше глубже объект заходит внутрь своего класса (**высокая уверенность**)
## Функционал потерь
### MSE
- Можем расписать ошибку через MSE, тогда: 
	- $$L_{mse} = (y_{i} - x_{i}^{T}w)^{2}=\frac{((y_{i})^{2}-y_{i} \dot x_{i}^{T}w}{y_{i}^{2}}=(1 - y_{i} x_{i} w)^{2}= (1 - M_{i})^{2}$$ ![[Pasted image 20260228114913.png]]
	- Минус состоит в том, что мы штрафуем модель за то что она лежит в толще собственного класса
### Функция sign
- В обычной линейной классификации мы используем ее как функционал потерь
- Но в данном алгоритме она не подойдет, по нескольким причинам
	- Нельзя использовать градиентный спуск
	- Функция не говорит на сколько модель уверенная в ответе
	- Функция постоянная имеет резкие скачки
### Функция сигмоиды
- Это уже сглаженная функция sign, которая переводит числа из рациональных в рационально ограниченные: $\sigma (z) = \frac{1}{1 + e^{-z}}; \sigma: (-\infty; +\infty) \rightarrow (0, 1)$ 
- Зачем необходима:
	- **Дифференцируемость**: У нее отличная производная, что позволяет обучать модель градиентным спуском
	- **Вероятностная интерпретация**: Можно использовать результат сигмоиды как вероятность. Например если на выходе мы получим значение 0.8, то модель на 80% уверенна в положительном ответе
- **Свойства:**
	- $1 - \sigma(z) = \sigma(-z)$
	- $\sigma(z)' = \sigma(z)(1 - \sigma(z))$ 
### Log Loss
- Если мы предсказываем вероятности, то мы не можем использовать обычные функции потерь
	- Разница между вероятностями будем слишком низкой, поэтому на ответах, где модель ошибается на 0.8 будут считаться небольшими ошибками
- Поэтому на необходим функционал, который будет штрафовать при отклонении на 0.1 или 0.5
- Но практически не будет на ошибке в 0.01
- $$L(y, p) = -(y \log(p) + (1 - y)\log(1 - p)), \text{где y - реальный ответ, а p - ответ модели}$$ 
- Log Loss — это фактически гладкая аппроксимация пороговой функции потерь (индикатора ошибки)
## Вероятностная модель
- Вероятностный подходи рассматривает модель не просто как дискретную функцию, а как модель апостериорного распределения классов $P(y|x)$
- Согласное **[[MLE|принципу максимального правдоподобия]]**, оптимальный вектор весов $w$ должен максимизировать вероятность наблюдения имеющейся выборки $X^{l}$
	- Для независимых **испытаний Бернулли** (*испытания, в которых существуют только два исхода*) функция правдоподобия представляет собой произведение индивидуальных вероятностей  $$L(w) = \prod\limits_{i=1}^{l}P(y_{i}|x_{i};w)$$
	- Можешь превратить формулу при помощи логарифмического правдоподобия для упрощения дифференцирования $$\ln L(w) = \sum\limits_{i=1}^{l}{\ln \sigma(M_{i)}}= \sum\limits_{i=1}^{l}\ln {\frac{1}{1 + e^{-M}}} = -\sum\limits_{i=1}^{l}\ln(1 + e^{-M})$$
- Минимизация эмпирического риска эквивалента максимизации согласия модели с данными. 
	- То есть нахождение такой плоскости, которая не только *разделяет классы* но и *делает проценты наиболее вероятными*

## Геометрическое значение
- Можно представить линейную регрессию, как гиперплоскость, которая разделяет пространство $\mathbb{R}^{k}$
- Для данной плоскости у нас имеется нормаль, представимая как веса $w$
- При скалярном перемножении $\langle x_{i},w \rangle$ получаем значение, которое определяет угол, между $x$ и нормалью $w$
	- Поэтому если значение отрицательно, то конечная точка x лежит по одну сторону от гиперплоскости
	- А если положительно, то лежит по другую 
## Метрики
### Accuracy
- **Accuracy** - доля правильных ответов
	- $$Accuracy = \frac{1}{n}\sum\limits^{n}_{i=1}[y_{i} = \bar y_{i}]$$ 
- *Используется для сбалансированных данных*, те количество классов примерно одинаковое
	- Иначе может выйти модель, которая выдает большой accuracy при несоразмерно большом количестве определения одного класса
- **Balanced accuracy** - доля правильных ответов, в толще собственного класса
	- $$\text{Balanced accuracy} = \frac{1}{C} \sum\limits^{C}_{i=1}{\frac{\sum\limits_{i}{y_{i}= k \& \bar y_{i}=y_{i}}}{\sum\limits_{i}{y_{i}=k}}})$$
### Precision&Recall
- **Precision** - точность
- **Recall** - полнота
	- $$\text{Precision}=\frac{TP}{TP + FP}, \text{Recall} = \frac{TP}{TP+FN}$$
	- Возьмем в пример бинарную классификацию, относительно положительного класса
	- *TP* - это правильно предсказанные положительные объекты
	- *FP* - это предсказанные, негативные объекты
	- *FN* - это не пересказанные классы, положительного объекта
	- *TN* - это не предсказанные, негативные объекты
	  То есть, TF - это правильно ли мы определили подходит или не подходит под класс объект
	  А PN - это уже по настоящему ли объект относится к нашему классу
- **Работает только в толще собственного класса** 
#### F1 score
- **F1 score** - гармоническое среднее *precision* и *recall*
- $$F_{1} = \frac{2}{precision^{-1} + recall^{-1}} = \frac{precision * recall}{precision + recall}$$
- $$F_{\beta} = (1 = \beta^{2}) \frac{precision * recall}{\beta^{2}precision + recall}$$
	- Чем больше $\beta$, тем важнее recall
	- Чем меньше $\beta$, тем важнее precision
### Receiver Operating Characteristic(Roc)
- **ROC** - показывает соотношение между долей верно определенных положительных объектов и долей ложно положительных объектов
- $$FPR = \frac{FP}{FP + TN} = 1 - Recall_{negative}, TPR = \frac{TP}{TP +FN}$$
	- Сортируем по по возрастанию вероятность принадлежности класса и составляем кумулятивные суммы
	- Проходя по выборке, меняем значения FPR и TPR 
	- 
- *Работает практически AP в obj detection *
- *Смотрим на площадь над кривой*,если больше 0.5, то это уже хорошо
	- Если меньше 0.5, то модель хуже случайного распределения классов
