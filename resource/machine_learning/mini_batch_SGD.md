# Mini-Batch SGD

## 📝 Основное содержание
- Метод градиентного спуска, который обновляет веса не по одному примеру (SGD) и не по всему датасету (Batch GD), а по небольшому случайному подмножеству — батчу

## Три режима градиентного спуска

| Метод | Размер батча | Обновление на шаге |
|---|---|---|
| Batch GD | Весь датасет | Точный градиент, но медленно |
| SGD | 1 пример | Шумный градиент, но быстро |
| **Mini-Batch SGD** | $m$ примеров | Компромисс |

---
## Алгоритм
На каждом шаге:
1. Случайно выбрать батч $\mathcal{B} = \{x_1, ..., x_m\}$ из датасета
2. Посчитать градиент по батчу:
$$g_t = \frac{1}{m} \sum_{i=1}^{m} \nabla L(x_i, \theta)$$
3. Обновить веса:
$$\theta_{t+1} = \theta_t - \alpha \cdot g_t$$
- Обозначения:
$$\begin{aligned}
&m \text{ — размер батча (batch size)} \\
&\alpha \text{ — learning rate} \\
&g_t \text{ — усреднённый градиент по батчу} \\
&\mathcal{B} \text{ — батч (подмножество датасета)}
\end{aligned}$$
---
## Эпоха (Epoch)

**Эпоха** — один полный проход по всему датасету. За одну эпоху делается $\frac{N}{m}$ шагов обновления, где $N$ — размер датасета.

---

## Влияние размера батча

| Маленький батч | Большой батч |
|---|---|
| Шумный градиент | Точный градиент |
| Быстрее обновления | Медленнее обновления |
| Лучше обобщает (регуляризующий эффект шума) | Хуже обобщает |
| Меньше памяти GPU | Больше памяти GPU |

> Типичные значения: 32, 64, 128, 256

## Связь с другими оптимизаторами

> ⚠️ Mini-Batch SGD — это не отдельный оптимизатор, а **режим подачи данных**. Adam, SGD с моментом и все остальные оптимизаторы по умолчанию работают именно в mini-batch режиме.
