## Алгоритм
- Алгоритм DFS основан на стеке, *обходит ветку до конца вглубь*, пока не упрется, потом идет обратно
- Главная сила DFS в структурной информации о графе
### Для чего используется
- Помогает определить является ли корнем узла другой узел. Необходима реализация на рекурсии при помощи с счётчиком времени.
	- Для каждой вершины появляется новое поле `tin` и `tout`, которая определяет время входа и выхода из рекурсии, таким образом, если $[tin[j], tout[j]] \subset [tin[i], tout[i]]$, то узел j является дочерним к j
- Поиск циклов: если повторно заходим в посещенный узел, то имеется цикл
### Псевдокод
- Реализуется либо на стеке, либо на рекурсивном обходе
- *Рекурсивная версия*:
```C++
vector<bool> visited(n, false);

void dfs(int u, vector<vector<int>>& adj) {
    visited[u] = true;
    for (int v : adj[u]) {
        if (!visited[v]) {
            dfs(v, adj);
        }
    }
}
```
- *Стековая версия*: 
```C++
void dfs_iter(int s, vector<vector<int>>& adj) {
    vector<bool> visited(n, false);
    stack<int> st;
    st.push(s);

    while (!st.empty()) {
        int u = st.top(); st.pop();
        if (visited[u]) continue;
        visited[u] = true;
        for (int v : adj[u])
            if (!visited[v])
                st.push(v);
    }
}
```
### Классификация узлов
- *Tree edge* - color\[v\] \=\=0
	- Если не посетили ребро за DFS
- *Back edge* - color\[v\] \=\= 0
	- Если уже постили ребро в v
- *Forward edge* - colo\[v\] \=\= 2 && tin\[u\] < tin\[v\]
	- v - потомок, u - не в дереве
- *Cross edge* - color\[v\] \=\=2 && tin\[u\] > tin\[v\]
	- Нет отношений предок или потомок
>color\[v\] - это позиция узла, 0 - значит, что в узел еще не заходили, 1 - значит мы рекурсивно в нем, 2 - значит что мы уже его прошли
### Асимптотика
- Память: $O(n + m)$
- Время: $O(n + m)$
