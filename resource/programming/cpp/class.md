- Класс - агрегатный тип, который был введен на основе уже существующих типов 
# Определение типа
```cpp
class <ClassName>{
	<ElementList>
} [<exCls1>], ..., [<exClsN>];
// Создание экземепляров
<Class_Name> <exCls1>, ... , <exClsN>;
```
- \<ClassName\> - название класса
- \<ElementList\> - список элементов класса
- \<exCls1\>, ... \<exClsN> - имена экземпляров класса
- Классы могут определяться как в программе, так и непосредственно вместе с классом
## Альтернативная форма определения классов
```cpp
typedef class <Class_Name>
{
    <Element_List>
} <Alias_Name>;
```
- При таком определении класс может иметь несколько названий
## Модификаторы доступа
- `public` - доступ возможен из любой части кода
	- Только необходимое нужно помещать в public часть класса, чтобы пользователь не мог изменять или просматривать важные данные
- `private` - доступ возможен только из внутренних методов класса
	- Все внутренние элементы находятся тут
- `protected` - доступ возможен из внутренних методов класса и его потомков
	- Помещаются те методы и поля, которые не должны быть видны снаружи, но реализация которых может быть переопределена и использована в потомках класса
```cpp
class Example
{
    private:
	    <Elem_list_1>
    public:
	    <Elem_list_2>
	protected:
		<Elem_list_3>
}; 
```
## Компоненты класса
- Поля
- Функции
- Классы
- Структуры
- Перечисления
- Битовые поля
### Поля
- Поля содержат всю необходимую информацию о классе,
- Изменение состояния объекта связанно с изменением его полей
- Поля класса могут иметь любой тип данных, кроме этого же типа( исключения ссылки и указатели данного класса ) 
- При описании полей их инициализация не допускается
```cpp
[<specifier>] [<cv_qualifier>] <type> <field_name> 
```
- \<specifier\> - спецификатор существования (static);
- \<cv-qualifier\> - квалификатор изменчивости(const);
- \<type\> - тип поля класса(int, double);
-  \<field_name\> - идентификатор поля класса(data, str, value);
#### Статическое поле
- *Статическое поле* - поле класса, которое определяется для всех его экземпляров
- `static int count`
- Память для статического поля выделяется всего раз и для всех экземпляров
- Память для стат класса не учитывается при исполнении `sizeof()`
- Изменяется стат поле либо обращение к пространству имен класса ( через экземпляр или без него ), либо статическими методами, 
### Методы
- Внутренняя функция класса
```cpp
<ret_value> <func_name>(<param_list>){
	<code_block>
}
```
- Класс может содержать один или более методов;
- Метод объекта – программный код, выполненный в виде процедуры или функции;
- Вызов метода объекта может приводить к изменению его состояния (значение полей-данных) или нет.
#### Статические методы
- *Статические методы* - методы, которые могут использовать в отсутствии экземпляров класса
- `static <ret_type> <func_name>(<param_list>){<code_block>};`
- Статические методы могут вызывать только другие статические методы
- Статические методы не имеют доступа к указателю `this`

# Конструктор класса
- *Конструктор класса* - метод, который вызывается при создании объекта
```cpp
class <ClassName>{
	<ClassName>([<FormalList>]){
		<CodeBlock>
	}
}
```
- Конструктор имеет то же имя что и сам класс
- Не возвращает никакого значения 
- Нельзя получить указатель на конструктор 
- Класс может иметь несколько конструкторов с разными параметрами для разных способов инициализации 
- Конструктор, вызываемый без параметров, называется конструктором по умолчанию
- Может иметь значения по умолчанию
- Нельзя описывать с модификаторами const, virtual и static
- Конструктор вызывается в момент создания объекта
- Конструкторы глобальных объектов вызываются до вызова функции main
```cpp
class <ClassName>{
	<ClassName>([<FormalList>]) : [<InitList>]{
		<CodeBlock>
	}
}
```
- \<ClassName\> - название класса;
- \<FormalList\> - список формальных параметров конструктора;
- \<InitList\> - список инициализаторов вида:
## Деструктор класса
- Деструктор - метод класса, который вызывается при разрушении объекта
- Не имеет аргументов
- Не может быть перегружен
- Не имеет возвращаемого значения
- Не может быть объявлен как const и static
- Может быть виртуальным
### Вызов деструктора
- Деструктор вызывается: 
	  - При выходе из блока кода, в котором он был объявлен
	  - Для глобальных объектов - как часть процедуры выхода из программы
	  - Для динамических объектов при использовании операции delete delete\[\]
# Внутренние классы
- Внутренний класс(nested) - вложенный класс, имеющий область действия ограниченную внешним классом
```cpp
class Parent{
	class [<NestedName>]{
		<ElementList>
	};
};
```
- Внутренний класс нельзя использовать вне области видимости родительского, поэтому его можно вызывать либо в методах родительского класса, либо с использованием пространства имен ( если позволяет модификатор доступа )
- При этом, дочерний класс не имеет доступа к полям и методам класса родителя
- Не имеют статичных данных ( в зависимости от стандарта, начиная с C++11 )
- Одно из применений - это создание исключений класса
```cpp
class Parent {
public:
	class Exception {
	public:
		char *Msg;
		Exception(char *theMsg) : Msg(theMsg) { }
	};
	//---------------------------------------------
	class PrintException {
	public:
		char *Msg;
		PrintException(char *theMsg) : Msg(theMsg) {}
	};
	//---------------------------------------------
};

Void Call() {
	try {
		Parent ex1;
		ex1.callPrintException();
	}
	catch(Parent::PrintException ex) {
		cout<<ex.Msg;
	}
	catch(Parent::Exception ex) {
		cout<<ex.Msg;
	}
}
```
