# CVE и БДУ ФСТЭК
- Перечни уязвимостей
- БДУ ФСТЭК - Российский перечень, попадают уязвимости характерные для России
- В CVE уязвимость хранится в таком формате `"CVE-<год_нахождения>-<код_уязвимости"` после чего идет описание уязвимости и показатель уязвимости `CVSS:<уровень_уязвимости>` ( чем выше уровень, тем выше опасность )
	- CVSS - то открытый стандарт, который используется для оценки степени опасности и серьезности уязвимостей в компьютерных системах
	- Имеет какие то метрики, связанные со многими векторами, с какими данными связан
	- НАДО ЗАПИСАТЬ: что такое, какие метрики юзает, чуть чуть вглубь
- CWE ( перечень общих слабостей ) - это система категории слабых мест и уязвимостей АО и ПО
	- Для каждой уязвимости свой идентификатор
	- Просто набор уязвимостей, без конкретного уровня опасности
	- Есть описания, примеры визуальные и при помощи кода на разных ЯП
- Пример слабости - не ограниченный доступ к использованию буфера памяти ( можно выйти за рамки буфера и получить доступ ко всей оперативной памяти )
- Файл подкачки - файлы, которые пока еще не нужны, но могут пригодиться в ближайшее время ( например, когда приложения находятся на фоне )
	- Файлы подкачки должны быть либо равны объему оперативной памяти либо ее удвоенному размеру
- [!] Атаки приводят к слабостям, слабости приводят к атакам
- [!] CVE -> CWE -> CAPEC 
## Пример слабости
- CWE-863 приводит к 5 атакам
	- Спуфинг
	- Обход авторизации
	- Переполнение буфера
	- Вызов микросервисов через не доверенный регистратор

# The Cyber Kill Chain
 - Модель, описывающая этапы, через которые проходит кибератака, от разведки до достижения цели. 
## Первый этап. Разведка
- Состоит из сбора информации: сбор E-mail, просмотр социальных сетей, пассивный поиск, определение IP, сканирование портов
	- Цель хакеров: найти как можно больше информации
	- Цель кибербезопасника: сделать как пространство для разведки как можно меньше
## Второй этап. Вооружение
- Состоит из создания вредоносного кода, продумывание системы доставки,  продумывание приманки. 
## Третий этап. Доставка
- Для доставки могут использоваться различные методы:
	- Фишинг - попытка получить ценные данные, под видом доверенного источника: банк, социальная сеть, магазин. 
	- Зараженные сайты - 
	- Операторы связи
## Четвертый этап. Проникновение
 - Состоит из активации, исполнении кода, определение плацдарма, проникновение на 3RD ресурсы. 
## Пятый этап. Инсталяция
- Уже на этом этапе устанавливаются троян или backdoor
	- Пытаются повысить права в системе
	- Использование руткитов
	- Обеспечивается незаметность
