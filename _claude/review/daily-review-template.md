# Daily Review Template

> Этот файл — шаблон для `_claude/reviews/YYYY-MM-DD-review.md`.
> Claude заполняет автоматически на основе Calendar + Daily diary.
> Vildan не пишет сюда вручную.

---

## Шаблон: `_claude/reviews/YYYY-MM-DD-review.md`

```markdown
---
date: YYYY-MM-DD
weekday: Понедельник / ... / Воскресенье
type: daily-review
study_hours_actual: 0
study_hours_planned: 0
---

# Ревью: DD Month YYYY

## Факты дня

**Запланировано в Calendar:**
- HH:MM – HH:MM | [Название события] | [Приоритет A/B/C/D]
- ...

**Выполнено:**
- ✅ [событие] — [краткий результат]
- ❌ [событие] — [причина срыва]

**Итого:** [X] ч учёбы из [Y] запланированных

---

## Прогресс по roadmap

**Тема:** [название]  
**Статус:** В работе / Завершено / Заблокировано  
**Детали:** [что именно прошёл, какие формулы/концепты]

**Созданные конспекты:** [[ссылка]] или "нет"

---

## Корректировки на завтра

**Перенесено:**
- [задача] → [новый слот]

**Добавлено:**
- [задача] | [слот] | [причина]

**Отменено:**
- [задача] | [причина]

---

## Флаги Claude

> Здесь Claude фиксирует наблюдения без запроса.

- ⚠️ [риск / паттерн / противоречие]
- 📌 [важное наблюдение для roadmap]
- 🔁 [систематическая проблема]

---

## Log-запись

Добавить в `_claude/log.md`:
`## [YYYY-MM-DD] review | Daily review — [краткая оценка дня 3-5 слов]`
```

---

## Правила заполнения

1. **study_hours_actual** — считай только реальные учебные блоки из Calendar (не работу, не gym).
2. **Факты** — только события из Calendar + Daily diary. Не додумывай.
3. **Прогресс по roadmap** — только если был реальный учебный контент.
4. **Флаги** — пиши всегда, даже если всё хорошо (тогда "Флагов нет").
5. **Не создавай файл** если нет ни Daily diary, ни учебных событий.

---

## Пример заполненного ревью

```markdown
---
date: 2026-05-12
weekday: Вторник
type: daily-review
study_hours_actual: 2.5
study_hours_planned: 3
---

# Ревью: 12 мая 2026

## Факты дня

**Запланировано:**
- 06:30 – 07:00 | Flashcards SQL | A
- 20:00 – 22:30 | A/B тесты: дизайн эксперимента | A

**Выполнено:**
- ✅ Flashcards SQL — 20 карточек, оконные функции
- ✅ A/B тесты — прошёл расчёт sample size, пример с конверсией
- ❌ 22:30 – 23:00 блок повторения — устал после работы

**Итого:** 2.5 ч из 3 запланированных

## Прогресс по roadmap

**Тема:** A/B Testing — Sample Size & Power  
**Статус:** В работе  
**Детали:** Формула n = (z_α/2 + z_β)² · 2σ² / δ². Пример с CTR 5%→6%, power=0.8.

**Созданные конспекты:** [[Resources/sql/window-functions]]

## Корректировки на завтра

**Перенесено:**
- Повторение A/B → 06:30 среды

**Добавлено:** ничего

## Флаги Claude

- ⚠️ Третий день подряд срывается вечерний блок после 22:00. Рекомендую сдвинуть учёбу на 19:30–22:00.
- 📌 SQL flashcards идут хорошо — можно добавить 10 мин/день без ущерба.
```
