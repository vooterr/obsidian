# Что это
- CPython - реализация python на языку C, его интерпретатор
	- Считается одной из самых популярных и самой эталонной версией интерпретатора
## Чем отличается от питона
- Python - это *язык программирования*, а *CPython* - его *интерпретатор*
	- На самом деле отличие колоссальное
	- Если язык - это набор правил и синтаксиса, то интерпретатор - это программа, которая исполняет код на этом языке
	- Можно это сравнить как язык *C* и *gcc*
- **Вкратце, на Python вы пишете код, а CPython - отвечает за исполнение этого кода**

## Как компилируется
- При компиляции, код проходит через три этапа
	1. Инициализация интерпретатора
	2. Компиляция исходного кода в байт-код
	3. Исполнение байт-кода
###  1. Инициализация интерпретатора
- На этом этапе CPython готовит все необходимое: 
	- Выделяет основные структуры данных ( int, str )
	- Загружает модули
	- Настраивает система импорта 
### 2. Компиляция исходного кода в байт-код
- Несмотря на на то что, CPython называют интерпретатором, все не так просто
	- Вначале код разбирается лексером на токены
	- После чего токены парсятся и строится синтаксическое дерево
	- Впоследствии дерево переводится в байт-код (именно байт, не машинный)
- Результатом компиляции будет объект кода (code object)
	- Содержащий байт-код, модули и функции
- Обычно код сохраняется с расширением .pyc в pycache ( чтобы не строить код по новой )
### 3. Исполнение байт-кода
-  Байт-код передается на исполнение виртуальной машине CPython 
- Машина читает данный код и исполняет его в код на уровне C++ кода

## Управление памятью
- В CPython используется комбинированный подход к управлению памятью 
	- Используется *подсчет ссылок*
	- И циклический сборщик мусора
### Сборщик мусора
- Каждый объект в Python имеет свой подсчет ссылок ( он считает ссылки которые указывают на него )
	- При создании нового объекта, число его ссылок становится равно 1
	- Если создать еще одну переменную на данный объект, счетчик ссылок увеличивается
	- Если число ссылок становится равно 0, данный объект удаляется
### Циклический сборщик мусора
- Однако иногда появляется проблема, что два python-структуры указывают друг на друга, тогда их количество ссылок никогда не опустится до нуля
- Чтобы решить данную проблему в Python существует циклический сборщик мусора
	- Это объект, который периодически просматривает объекты python и определяет группы объектов, ссылающихся друг на друга, но при этом не имеющих внешних ссылок
	- При обнаружении таких групп, сборщик мусора будет их уничтожать
- Также стоит отметить, что объекты в питоне *делятся на поколения по возрасту*
	- Всего существуют 3 поколения (0, 1, 2)
		- В нулевом поколении находятся самые молодые объекты
		- Во 2 втором самые старые
	- В зависимости от поколения, изменяется и периодичность проверки на циклические ссылки
	- Также существует отдельное поколение для статичный объектов, проверка на которые никогда не производится
### Выделение памяти
- В CPython находится специальный менеджер памяти под названием pymalloc
- Он специально оптимизирован под выделение памяти для маленьких объектов (<= 512 байт)
	- Вместо того, чтобы использовать malloc для каждого объекта pymalloc резервирует большие куски памяти (256 КБайт) и разделяет их на пулы (4 КБайт), а затем их на блоки под объекты нужного размера
- Для всего что больше 512 Байт pymalloc делегирует задачу malloc
- CPython пытается оптимизировать память, поэтому он интернирует(кэширует) некоторые объекты памяти
	- Например, небольшие числа от -5 до 256 являются singleton объектами
	- То есть, если инициализировать два объекта числами 256, то они будут указывать на один и тот же объект
- Аналогично CPython интернирует некоторые строки, как минимум похожие на идентификаторы (именованные переменные) и короткие строки
# GIL глобальная блокировка интерпретатора 
## Что такое GIL
- GIL (gloabal interpreter lock) - глобальная блокировка интерпретатора
	- Блокировка, которая позволяет одному потоку выполнять один байт-код даже на многоядерном процессоре
- Он был создан для безопасного вычисления байт кода в python
	- Операции с ссылками не являются атомарными
		- Дело в том, что постоянные изменения могут привести к неявным результатам
		- Допустим у нас есть два процесса, которые одновременно увеличивают количество ссылок, или один уменьшает другой увеличивает, или вообще один удаляет объект, а другой создает
	- Таким образом, в многопотоке очень сложно реализовать безопасный способ решения данной задачи.
